Move the duplicated `STDLIB_CRATES` constant and `is_stdlib_crate()` function to a single shared location.

## Problem

`const STDLIB_CRATES: &[&str] = &["std", "core", "alloc"]` is defined identically in:
- `src/resolve.rs:9`
- `src/stdlib.rs:8`

Each file also has its own `is_stdlib_crate()` function.

## Fix

1. Add to `src/types.rs`:
   ```rust
   pub(crate) const STDLIB_CRATES: &[&str] = &["std", "core", "alloc"];

   pub(crate) fn is_stdlib_crate(name: &str) -> bool {
       STDLIB_CRATES.contains(&name)
   }
   ```
2. Remove the constant and function from `src/resolve.rs` and `src/stdlib.rs`
3. Update all call sites to use `crate::types::is_stdlib_crate` or `use crate::types::STDLIB_CRATES`
