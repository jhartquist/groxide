Move render dispatch logic from lib.rs into the render module.

## Problem

lib.rs contains ~200 lines of render dispatch logic that belongs in the render module:
- `handle_output()` (~87 lines) — dispatches Found/Ambiguous/NotFound to renderers
- `render_recursive()` (~18 lines) — recursive mode dispatch
- `render_recursive_source()` (~27 lines) — recursive source mode
- `render_impls()` (~15 lines) — impl rendering dispatch

## Fix

1. Create `src/render/dispatch.rs`
2. Move the four functions listed above from `src/lib.rs`
3. Add `pub(crate) mod dispatch;` to `src/render/mod.rs`
4. Update call sites in lib.rs to use `render::dispatch::` prefix
5. These functions depend on: `render::*` submodules, `types::*`, `query::is_reexport_stub`, `reexport::*` (from task 77)

## Important

- Do NOT change any function signatures or behavior
- Do NOT modify tests
- Keep `pub(crate)` visibility on moved functions
- This can be done in parallel with task 77 (reexport.rs) but may need to coordinate on imports if reexport.rs is also being extracted
