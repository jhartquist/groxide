## Workspace-wide querying: detect virtual manifest and list all crates

**Plan reference:** docs/plans/2026-03-05-cli-redesign-implementation.md — Phase 4, Task 13

When no path argument is given and CWD is at a virtual workspace root, detect all workspace members and show each crate with its top-level children (same as running `grox <crate>` for each member).

### What to do

1. **Add workspace introspection methods** to `ProjectContext` in `src/resolve.rs`:

```rust
/// Returns true if the workspace has no root package (virtual manifest).
pub(crate) fn is_virtual_workspace(&self) -> bool {
    self.metadata.resolve.as_ref()
        .and_then(|r| r.root.as_ref())
        .is_none()
}

/// Returns all workspace member packages, sorted by name.
pub(crate) fn workspace_member_packages(&self) -> Vec<&Package> {
    let mut members: Vec<_> = self.metadata.workspace_members.iter()
        .map(|id| &self.metadata[id])
        .collect();
    members.sort_by_key(|p| &p.name);
    members
}
```

2. **Add workspace detection in `run()`** in `src/lib.rs`:
   Before the existing query path parsing, add:

```rust
if cli.path.is_none() && ctx.as_ref().is_some_and(|c| c.is_virtual_workspace()) {
    return handle_workspace(&mut out, ctx.as_ref().unwrap(), cli);
}
```

3. **Implement `handle_workspace` function** in `src/lib.rs`:
   - Iterate over all workspace member packages
   - For each, create a CrateSource::CurrentCrate, call load_or_build_index
   - Resolve the crate root in the index
   - Render based on current detail tier flags:
     - Default: crate name header + top-level children with signatures (like default crate view)
     - -b: crate name + child names only
     - -d: crate name + children with full docs
     - -r: recursive tree for all crates
     - -r -b: recursive names only for all crates
     - -r -s: recursive with source for all crates
   - Separate each crate with a blank line

4. **Expected default output format:**
```
rloop_core
  mod    config
  mod    traits
  struct Task           fn new(name: &str) -> Self

rloop_db
  mod    memory
  mod    sqlite
  trait  Store          fn get(&self, id: &str) -> Result<Task>
```

5. **Write unit tests** for the new ProjectContext methods.

6. **Integration testing:** This is hard to test with the single-crate fixture. Options:
   - Create a workspace fixture under test-fixtures/ with 2-3 small crates
   - Or mark tests as `#[ignore]` for manual testing against real workspaces
   - At minimum, test that `is_virtual_workspace` returns false for a normal crate
