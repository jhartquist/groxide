## Show hint when search returns 0 results and crate has non-default features

**Plan reference:** docs/plans/2026-03-05-cli-redesign-implementation.md — Phase 5, Task 15

When an agent searches for "viridis" in colorgrad and gets 0 results, it wastes 3 calls discovering --all-features. Show a hint: "hint: N items found with --all-features"

### What to do

1. **Add a feature-gated item to the fixture crate:**
   In `test-fixtures/groxide_test_api/src/lib.rs`:
   ```rust
   /// Unstable API for testing feature gates.
   #[cfg(feature = "unstable")]
   pub fn unstable_api() -> bool {
       true
   }
   ```
   The fixture already has `[features] unstable = []` in its Cargo.toml.

2. **Implement hint in `handle_search`** in `src/lib.rs:391-460`:
   After the search returns 0 results, check if the crate has non-default features:
   
   - Check `features.all_features` — if already true, don't hint
   - Create a new FeatureFlags with `all_features: true`
   - Rebuild the index with all features (expensive but only on 0 results)
   - Re-run the search
   - If the new search finds results, output to stderr:
     ```
     hint: {count} items found with --all-features
     ```
   - The stdout output remains "0 results for ..." (don't change the exit behavior)

3. **Need access to source info** to determine if crate has features. Options:
   - Pass the CrateSource to handle_search
   - Check if the source is CurrentCrate or Dependency (they have manifest_path) and read Cargo.toml features
   - Or simpler: always try the all-features rebuild on 0 results if not already using all-features. If the rebuild finds the same 0 results, don't show the hint.

4. **Write integration test** in `tests/cli_happy.rs`:
   ```rust
   #[test]
   fn search_zero_results_hints_features() {
       let output = grox()
           .args(["-S", "unstable_api", "groxide_test_api"])
           .output()
           .expect("command runs");
       let stdout = String::from_utf8_lossy(&output.stdout);
       let stderr = String::from_utf8_lossy(&output.stderr);
       assert!(stdout.contains("0 results"));
       assert!(stderr.contains("hint:") && stderr.contains("--all-features"));
   }
   ```

5. **Performance consideration:** The hint rebuilds the index with all features, which is expensive. Only do this on 0 results. For local crates this is fast (cargo incremental). For external crates it could be slow — consider skipping the hint for external crates or caching the result.
