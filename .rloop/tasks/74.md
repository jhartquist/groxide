Remove the serialize-then-deserialize roundtrip hack used to copy DocIndex for cache storage.

## Problem

`src/cache.rs:144-151` contains `serialize_index_ref()` which serializes a `&DocIndex` to msgpack bytes then immediately deserializes it back — a full O(n) binary roundtrip just to produce an owned copy. This exists because `CachedData` needs to own its `DocIndex` and `DocIndex` doesn't derive `Clone`.

## Fix

1. Add `#[derive(Clone)]` to `DocIndex` in `src/types.rs`. Also add Clone to any field types that need it (e.g., `IndexItem`, `TraitImpl`, `ChildRef`).
2. Delete the `serialize_index_ref()` function from `src/cache.rs`
3. Replace the call site in `save_to_cache()` with `index.clone()`
4. Remove the doc comment explaining the hack
