## Add new detail tier flags to CLI

**Plan reference:** docs/plans/2026-03-05-cli-redesign-implementation.md — Phase 1, Task 2

Add two new flags that control how much detail is shown per item. These form a strict hierarchy with the existing -s flag:
- -b/--brief: kind + name only (compact)
- (default): kind + name + signature
- -d/--docs: kind + name + signature + full rendered docs
- -s/--source: kind + name + signature + rendered docs + source code

### What to do

1. **Add fields to `Cli` struct** in `src/cli.rs`:

```rust
/// Show only item names (compact output)
#[arg(short = 'b', long, conflicts_with_all = ["docs", "source"])]
pub brief: bool,

/// Show full rendered documentation per item
#[arg(short = 'd', long, conflicts_with_all = ["brief", "source"])]
pub docs: bool,
```

2. **Update existing conflict lists:**
   - `source` field: add `"brief"` and `"docs"` to conflicts
   - `search` field: add `"brief"` and `"docs"` to conflicts
   - `impls` field: add `"brief"` and `"docs"` to conflicts

3. **Write integration tests** in `tests/cli_errors.rs`:
   - Test that `--brief` and `--docs` conflict with each other
   - Test that `--brief` and `--source` conflict
   - Test that `--docs` and `--source` conflict
   - Test that `--brief` alone succeeds (exit 0)
   - Test that `--docs` alone succeeds (exit 0)

4. At this point the flags exist but don't change behavior yet — that comes in Phase 2 tasks.
