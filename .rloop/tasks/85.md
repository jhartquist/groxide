Make feature gate suffix formatting use a single source of truth across all renderers.

## Problem

Feature gate formatting is scattered across multiple render files:
- `render/mod.rs:538` defines `feature_gate_suffix()` helper
- `render/text.rs:258-270` re-implements suffix composition in `render_name_line` and `render_signature_line`
- `render/list.rs` and `render/ambiguous.rs` each have their own inline composition

Changes to the format (e.g., changing `[feature = "..."]` to `#[cfg(feature = "...")]`) require updating 3+ places.

## Fix

1. Keep `feature_gate_suffix()` in `render/mod.rs` as THE single source of truth
2. Update `render/text.rs` to call `feature_gate_suffix()` instead of re-implementing the logic inline
3. Update `render/list.rs` to use the shared function
4. Update `render/ambiguous.rs` if it has its own inline implementation
5. Ensure the suffix is appended consistently (same spacing, same format) everywhere

## Important

- Read each usage site carefully before changing — some may have slightly different formatting needs
- Snapshot tests will catch any unintended output changes
