Merge two near-identical path computation functions in index_builder.rs.

## Problem

`src/index_builder.rs` has two functions that are ~90% identical:
- `compute_impl_paths()` (lines ~188-216) — computes paths for methods inside impl blocks
- `compute_trait_item_paths()` (lines ~229-258) — computes paths for items inside trait definitions

Both: iterate all items → check if kind matches (Impl vs Trait) → check if path exists → look up parent → extract children → build child path as `parent::child_name`.

## Fix

1. Create a generic function:
   ```rust
   fn compute_descendant_paths(&mut self, items: &HashMap<Id, Item>, filter: impl Fn(&ItemEnum) -> Option<&[Id]>)
   ```
   where `filter` extracts child IDs from the relevant parent kind
2. Call it twice with different filters:
   - For impls: `|inner| if let ItemEnum::Impl(i) = inner { Some(&i.items) } else { None }`
   - For traits: `|inner| if let ItemEnum::Trait(t) = inner { Some(&t.items) } else { None }`
3. Delete the two original functions

## Important

- Read both functions carefully for subtle differences before merging
- There may be differences in how impl blocks vs traits determine the "parent name" — handle this in the filter or with an additional parameter
- All existing tests must pass unchanged
