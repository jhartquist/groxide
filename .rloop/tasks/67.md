## Final cohesion and consistency pass

**Plan reference:** docs/plans/2026-03-05-cli-redesign-implementation.md

This is the final task. After all features are implemented and docs are updated, do a thorough consistency and cohesion check. In the past, examples in docs have been broken or out of date.

### What to do

1. **Run EVERY example from HELP_EXAMPLES in `src/cli.rs`:**
   - Build the binary: `cargo build`
   - Run each example command from the help text against either:
     - The fixture crate (for groxide_test_api examples)
     - A real crate like serde or tokio (for external examples)
   - Verify each produces sensible output and exits 0
   - Fix any broken examples

2. **Run every example from README.md:**
   - Extract all code examples and shell commands from README.md
   - Run them and verify they work
   - Fix any that are broken or show outdated output

3. **Run every example from SKILL.md** at `skills/groxide/SKILL.md`:
   - Same process: extract, run, verify, fix
   - The SKILL.md is what AI agents read — it MUST be accurate

4. **Run every example from CLAUDE.md:**
   - Same process

5. **Cross-check flag documentation consistency:**
   - Compare the flag table in README.md with the actual `Cli` struct in `src/cli.rs`
   - Compare CLAUDE.md flag descriptions with the actual behavior
   - Compare SKILL.md guidance with actual CLI behavior
   - Ensure no references to removed flags (-l/--list, --all/-a)
   - Ensure all new flags (-b/--brief, -d/--docs, --clear-cache, --impls <TRAIT>) are documented everywhere

6. **Check error messages:**
   - Run `grox --help` and verify output is clean
   - Run `grox -b -d groxide_test_api` and verify the conflict error is clear
   - Run `grox -s -b groxide_test_api` and verify the conflict error is clear
   - Run `grox --clear-cache` and verify it works

7. **Verify search syntax is documented:**
   - Ensure `|` for OR and space for AND is mentioned in CLAUDE.md, SKILL.md, and README.md
   - Run example search commands to verify

8. **Check workspace docs:**
   - Ensure workspace mode behavior is documented in README.md
   - Describe what happens at workspace root vs inside a crate

9. **Check for stale snapshots:**
   - Run `cargo test` with no INSTA_UPDATE to verify all snapshots are current
   - Any failing snapshots mean the docs or tests are out of sync

10. **Smoke test the full workflow an AI agent would use:**
    ```bash
    # Orient: what's in this crate?
    grox groxide_test_api
    # Search for something
    grox groxide_test_api -S "add"
    # Drill into it
    grox groxide_test_api::add
    # Get source
    grox groxide_test_api::add -s
    # Brief overview
    grox groxide_test_api -r -b
    # Full dump
    grox groxide_test_api -r -s
    # Check impls
    grox groxide_test_api::GenericStruct --impls
    grox groxide_test_api::containers::Stack --impls Default
    ```
    Every command should work and produce useful output.
