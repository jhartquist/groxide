Bundle commonly-threaded render parameters into a context struct.

## Problem

Multiple render dispatch functions pass overlapping parameter sets like `(index, limits, mode, kind_filter, private)`. Adding a new parameter means updating function signatures across the render pipeline.

## Fix

1. Define in `src/render/mod.rs` or `src/render/dispatch.rs`:
   ```rust
   pub(crate) struct RenderContext<'a> {
       pub index: &'a DocIndex,
       pub limits: DisplayLimits,
       pub mode: OutputMode,
       pub kind_filter: Option<ItemKind>,
       pub include_private: bool,
   }

   impl<'a> RenderContext<'a> {
       pub fn from_cli(index: &'a DocIndex, cli: &Cli) -> Self {
           Self {
               index,
               limits: if cli.all { DisplayLimits::expanded() } else { DisplayLimits::default() },
               mode: cli.output_mode(),
               kind_filter: cli.kind.map(ItemKind::from),
               include_private: cli.private,
           }
       }
   }
   ```
2. Update render dispatch functions to accept `&RenderContext` instead of individual parameters
3. This is a gradual migration — start with the dispatch functions, then optionally propagate to individual renderers

## Important

- Do NOT force this into functions where only 1-2 parameters are needed — that adds unnecessary coupling
- The context struct should contain only parameters that are commonly passed together
- All existing tests must pass unchanged
