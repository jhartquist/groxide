Add a '## Security considerations' section near the bottom of README.md (before Contributing) that documents:

1. **Code execution**: groxide runs `cargo +nightly rustdoc` on crate source code. For your own project and its dependencies, this is the same trust model as `cargo build`. For auto-fetched external crates, groxide downloads source from crates.io and runs rustdoc on it — the same trust model as `cargo install`.

2. **Network access**: When auto-fetching, groxide contacts the crates.io API to resolve versions and download tarballs. No other network access occurs. Downloads are size-limited (500 MB) with timeouts.

3. **File system**: Index caches are written to `target/groxide/` (project-local) and `~/.cache/groxide/` (global). External crate tarballs are extracted with path-traversal protection.

Keep it factual and brief — 3 short paragraphs. Users of developer tools appreciate transparency about what the tool does on their system.
