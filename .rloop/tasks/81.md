Extract duplicated Levenshtein scoring logic into a shared function.

## Problem

`src/query.rs` has two near-identical functions:
- `compute_suggestions()` (~35 lines) — scores all index items
- `compute_method_suggestions()` (~30 lines) — scores a parent's children

Both do: prefilter with `could_match_within_distance()` → compute `levenshtein_distance()` → sort by (distance, name) → dedup → truncate to max.

## Fix

1. Extract a shared function:
   ```rust
   fn collect_suggestions(
       candidates: impl Iterator<Item = (String, String)>,  // (display_path, name)
       query: &str,
       max: usize,
   ) -> Vec<String>
   ```
2. Rewrite `compute_suggestions()` and `compute_method_suggestions()` to call `collect_suggestions()` with their respective iterators
3. Keep `levenshtein_distance()` and `could_match_within_distance()` as-is (they're shared utilities already)

## Important

- Preserve exact same suggestion output (same ordering, same dedup, same cap)
- All existing suggestion tests must pass unchanged
