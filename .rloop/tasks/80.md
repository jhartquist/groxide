Replace 4 separate exhaustive match blocks over ItemKind with a single const metadata table.

## Problem

`src/types.rs` has 4 match blocks over all 33 `ItemKind` variants:
- `short_name()` — returns display string like "fn", "struct", "trait"
- `matches_filter()` — checks if kind matches a KindFilter
- `category()` — maps to KindCategory for grouped display
- `is_primary()` — returns whether kind is a "primary" type

Adding a new variant requires updating all 4 match arms.

## Fix

1. Define a metadata struct:
   ```rust
   struct KindMeta {
       short_name: &'static str,
       category: KindCategory,
       is_primary: bool,
       filter: Option<KindFilter>,
   }
   ```
2. Create a const function or match that maps each ItemKind to its KindMeta
3. Rewrite `short_name()`, `category()`, `is_primary()`, `matches_filter()` as one-liners delegating to the metadata
4. Ensure all existing tests still pass

## Important

- The 33 variants must all be covered
- Do NOT use wildcard `_` matches — the compiler should catch new variants
- Keep the same public API (method signatures unchanged)
