Replace O(n²) Vec::contains() dedup with proper data structure.

## Problem

`src/error.rs:23-29` deduplicates suggestions using `Vec::contains()` in a loop — O(n²). While N ≤ 10, this is the wrong pattern.

## Fix

1. Replace with `HashSet` for seen-tracking while preserving insertion order:
   ```rust
   let mut seen = HashSet::new();
   let unique: Vec<_> = suggestions.into_iter()
       .filter(|s| seen.insert(s.clone()))
       .collect();
   ```
2. Or use `Vec::dedup()` after sorting (if order doesn't matter)
3. Preserve the existing behavior: first occurrence wins, cap at 5
