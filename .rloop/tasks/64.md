## Cache which crates fail with docs.rs metadata to skip on retry

**Plan reference:** docs/plans/2026-03-05-cli-redesign-implementation.md — Phase 5, Task 17

For crates like wgpu, the docs.rs metadata build fails, then retries with default features — doubling build time (20s). Cache these failures per (crate, version) so subsequent fetches skip straight to default features.

### What to do

1. **Find the docs.rs metadata fallback code:**
   Look in `src/docgen.rs` for where docs.rs metadata is attempted and where the fallback to default features happens. The stderr message is: `[grox] Build with docs.rs metadata failed, retrying with default features...`

2. **Create a simple failure cache file** at `~/.cache/groxide/docsrs-failures.json`:

   In `src/docgen.rs` (or a new helper module):
   ```rust
   use std::collections::HashSet;

   /// Records that docs.rs metadata build failed for a crate@version.
   fn record_docsrs_failure(name: &str, version: &str) {
       let cache_dir = dirs::cache_dir().unwrap_or_default().join("groxide");
       let path = cache_dir.join("docsrs-failures.json");
       let mut failures = load_docsrs_failures(&path);
       failures.insert(format!("{name}@{version}"));
       let _ = std::fs::create_dir_all(&cache_dir);
       let _ = std::fs::write(&path, serde_json::to_string(&failures).unwrap_or_default());
   }

   /// Checks if docs.rs metadata is known to fail for this crate@version.
   fn is_docsrs_known_failure(name: &str, version: &str) -> bool {
       let cache_dir = dirs::cache_dir().unwrap_or_default().join("groxide");
       let path = cache_dir.join("docsrs-failures.json");
       let failures = load_docsrs_failures(&path);
       failures.contains(&format!("{name}@{version}"))
   }

   fn load_docsrs_failures(path: &Path) -> HashSet<String> {
       std::fs::read_to_string(path)
           .ok()
           .and_then(|s| serde_json::from_str(&s).ok())
           .unwrap_or_default()
   }
   ```

3. **Wire into the build flow:**
   - Before attempting docs.rs metadata build, check `is_docsrs_known_failure`
   - If known failure, skip straight to default features build
   - After a docs.rs metadata build fails, call `record_docsrs_failure`

4. **Ensure --clear-cache also clears this file** (it already does since it removes the whole ~/.cache/groxide/ dir).

5. **Write unit tests:**
   - Test round-trip: record failure, check it's known
   - Test unknown crate returns false
   - Test file doesn't exist returns false
