Standardize field names in the CacheMetadata enum variants.

## Problem

In `src/cache.rs`, `CacheMetadata::Dependency` uses `package_version: String` while `CacheMetadata::External` uses `crate_version: String`. Same concept, different names.

## Fix

1. Rename both fields to `version: String`
2. Update all match arms in `is_cache_valid()` and `create_header()`
3. Bump `FORMAT_VERSION` constant by 1 so old caches are invalidated cleanly (they'll just rebuild)
