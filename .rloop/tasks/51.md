## Make --impls accept an optional trait name for filtering

**Plan reference:** docs/plans/2026-03-05-cli-redesign-implementation.md — Phase 1, Task 3

Currently --impls is a bare boolean flag that dumps ALL trait implementations. Change it to accept an optional trait name to filter results. `grox wgpu::Device --impls Clone` should answer "does Device impl Clone?" with a targeted result.

### What to do

1. **Change the `impls` field type** in `src/cli.rs:55-57`:

```rust
/// Show trait implementations, optionally filtered by trait name
#[arg(short = 'i', long, conflicts_with_all = ["source", "brief", "docs"],
      num_args = 0..=1, default_missing_value = "")]
pub impls: Option<String>,
```

2. **Update all references to `cli.impls`** in `src/lib.rs`:
   - Change `if cli.impls` to `if let Some(ref filter) = cli.impls`
   - Pass the filter value through to rendering:
   ```rust
   let trait_filter = if filter.is_empty() { None } else { Some(filter.as_str()) };
   ```

3. **Update `render_impls` function** in `src/lib.rs:769-782`:
   - Add `trait_filter: Option<&str>` parameter
   - Pass it to `render_impls_type`

4. **Add filtering to `render_impls_type`** in `src/render/ambiguous.rs`:
   - Add `trait_filter: Option<&str>` parameter
   - When set, filter `trait_impls` to only those whose `trait_path` contains the filter string (case-insensitive)
   - If no matching impls found, output "No implementation of '{filter}' found"

5. **Write integration tests** in `tests/cli_happy.rs`:
   - `impls_filter_by_trait_name`: query `groxide_test_api::containers::Stack --impls Default`, verify it shows Default impl and not others
   - `impls_filter_no_match`: query with a trait that doesn't exist, verify appropriate message
   - `impls_bare_flag_still_works`: query `groxide_test_api::containers::Stack --impls` (no value), verify all impls shown

6. **Update snapshots:** `INSTA_UPDATE=always cargo test`
