The README doesn't explain how groxide behaves differently inside vs outside a Rust project. Add a section after "How it works" (line ~126) called "## Project context" that explains:

1. **Inside a Rust project** (directory with Cargo.toml): groxide reads the project's dependency graph from Cargo.toml. `grox` with no arguments shows the current crate's docs. Queries resolve through: current crate -> direct deps -> workspace members -> transitive deps -> stdlib -> crates.io auto-fetch. The index cache lives in `target/groxide/`.

2. **Outside a Rust project**: Only stdlib queries (`grox std::collections::HashMap`) and crates.io auto-fetch (`grox serde::Deserialize`) work. Running `grox` with no arguments will error. The cache for external crates lives in `~/.cache/groxide/`.

Keep it concise — a short paragraph for each case. This helps users and LLM agents understand the tool's behavior.
