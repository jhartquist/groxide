Extract duplicated child ID collection logic in index_builder.rs into a helper function.

## Problem

`src/index_builder.rs` has 3 places that match `ItemEnum::Struct/Enum/Union/Trait` to extract child IDs:
- In `resolve_module_children()` — collects fields/variants for module listings
- In `pass4_link_relationships()` — links children to parent items
- In `resolve_use_children()` — resolves re-export targets

Each has a ~15-line match block doing essentially the same extraction.

## Fix

1. Create a helper function:
   ```rust
   fn get_item_child_ids(item: &Item) -> Vec<Id> {
       match &item.inner {
           ItemEnum::Struct(s) => struct_field_ids(s),
           ItemEnum::Enum(e) => e.variants.clone(),
           ItemEnum::Union(u) => u.fields.clone(),
           ItemEnum::Trait(t) => t.items.clone(),
           _ => Vec::new(),
       }
   }
   ```
2. Replace the 3 duplicated match blocks with calls to this helper
3. Adjust as needed — some call sites may need slightly different variant coverage

## Important

- Read each of the 3 call sites carefully before extracting — they may have subtle differences in which variants they handle
- All existing tests must pass unchanged
