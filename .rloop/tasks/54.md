## Implement brief rendering (kind + name only)

**Plan reference:** docs/plans/2026-03-05-cli-redesign-implementation.md — Phase 2, Task 6

Brief mode shows only kind + name. No signatures, no summaries, no docs. Works for both single-item default view and recursive mode.

### What to do

1. **Create `src/render/brief.rs`** with:
   - `render_brief(display: &DisplayItem) -> String` — for single-item/container view
   - `render_brief_recursive(items: &[&IndexItem], root_path: &str) -> String` — for recursive

The format should be clean and compact:
```
mod    containers
mod    traits
struct GenericStruct
enum   Direction
fn     add
```

For recursive, group by parent module (same as `render_list_recursive` structure):
```
groxide_test_api:
  mod    containers
  mod    traits
  struct GenericStruct
  fn     add

groxide_test_api::containers:
  struct Stack
  struct Pair
  const  MAX_CAPACITY
```

2. **Register the module** in `src/render/mod.rs`: `pub(crate) mod brief;`

3. **Wire into dispatch** in `src/lib.rs`:
   - In the recursive block (~line 702-717): check `cli.brief` first, call `render::brief::render_brief_recursive`
   - In the default output block (~line 730-741): check `cli.brief`, call `render::brief::render_brief`
   - Also handle the ambiguous case

4. **Write integration tests** in `tests/cli_happy.rs`:
   - `brief_mode_crate_root`: `grox -b groxide_test_api` — snapshot test
   - `brief_mode_module`: `grox -b groxide_test_api::containers` — snapshot
   - `brief_recursive_crate_root`: `grox -r -b groxide_test_api` — snapshot
   - Verify no signatures appear in output (no "pub fn", "pub struct" etc.)

5. **Write unit tests** in `src/render/brief.rs`:
   - Test with module DisplayItem
   - Test with type DisplayItem (should show method names)
   - Test recursive grouping

6. **Update snapshots:** `INSTA_UPDATE=always cargo test`
