## Remove conflicts between -r and detail flags

**Plan reference:** docs/plans/2026-03-05-cli-redesign-implementation.md — Phase 1, Task 5

Currently --recursive conflicts with --source, preventing the "dump everything" mode (-r -s). Remove these conflicts so all detail flags compose with -r.

### What to do

1. **Update conflict lists** in `src/cli.rs`:
   - `source` field: conflicts should be `["impls", "brief", "docs"]` — remove any recursive reference
   - `recursive` field: conflicts should be `["impls", "search"]` — remove "source", do NOT add "brief" or "docs" (they compose)
   - `brief` field: should NOT conflict with recursive
   - `docs` field: should NOT conflict with recursive

2. **Write integration tests** in `tests/cli_happy.rs`:

```rust
#[test]
fn recursive_and_source_composable() {
    let output = grox()
        .args(["-r", "-s", "groxide_test_api"])
        .output()
        .expect("command runs");
    // Should not fail with conflict error (exit 2)
    assert_ne!(output.status.code(), Some(2));
}

#[test]
fn recursive_and_brief_composable() {
    let output = grox()
        .args(["-r", "-b", "groxide_test_api"])
        .output()
        .expect("command runs");
    assert_ne!(output.status.code(), Some(2));
}

#[test]
fn recursive_and_docs_composable() {
    let output = grox()
        .args(["-r", "-d", "groxide_test_api"])
        .output()
        .expect("command runs");
    assert_ne!(output.status.code(), Some(2));
}
```

3. At this point the flag combinations are accepted but behavior may not be fully implemented yet — that comes in Phase 2 tasks. The tests just verify no conflict error.
