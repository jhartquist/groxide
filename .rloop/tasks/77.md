Extract re-export following logic from lib.rs into its own module.

## Problem

lib.rs contains ~150 lines of self-contained re-export logic mixed in with orchestration.

## Fix

1. Create `src/reexport.rs`
2. Move these functions from `src/lib.rs`:
   - `try_follow_reexport()` — follows cross-crate re-export stubs to canonical items
   - `parse_reexport_source()` — parses re-export source annotation from item docs
   - `annotate_reexport()` — annotates rendered output with re-export path info
3. Add `mod reexport;` to `src/lib.rs`
4. Update call sites in lib.rs to use `reexport::` prefix
5. These functions depend on: `cache`, `docgen`, `query`, `resolve`, `cli::FeatureFlags` — use `crate::` paths

## Important

- Do NOT change any function signatures or behavior
- Do NOT modify tests — they should still pass as-is
- Keep `pub(crate)` visibility on moved functions
