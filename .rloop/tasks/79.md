Extract source-reading logic from lib.rs into its own module.

## Problem

lib.rs contains source-reading logic that is a distinct concern from orchestration:
- `read_source_content()` (~35 lines) — reads source code from disk for any CrateSource variant
- `handle_source()` (~85 lines) — orchestrates single-item source mode output
- `handle_recursive_source()` — if present, the render portion goes to render/dispatch.rs (task 78)

## Fix

1. Create `src/source.rs`
2. Move `read_source_content()` and `handle_source()` from `src/lib.rs`
3. Add `mod source;` to `src/lib.rs`
4. Update call sites to use `source::` prefix
5. `read_source_content()` depends on `resolve::CrateSource` and std::fs — use `crate::` paths
6. `handle_source()` depends on `render::ambiguous::render_source`, `reexport::*`, and source reading

## Important

- Do NOT change any function signatures or behavior
- Do NOT modify tests
- Keep `pub(crate)` visibility on moved functions
