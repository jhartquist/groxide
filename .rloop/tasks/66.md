## Replace one-line summary with clean tier: signatures only in lists

**Plan reference:** docs/plans/2026-03-05-cli-redesign-and-workspace-support-design.md — Section 2

The current default for list/recursive mode shows a first-sentence summary truncated from docs. This is an arbitrary middle ground — it's neither full docs nor clean absence of docs. Per the design decision, the default tier should show kind + name + signature only (no summary). Full docs are available via -d.

### What to do

1. **Update `render_item_line` in `src/render/list.rs:112-149`:**
   - Remove the summary column from the default output
   - The line format becomes: `{kind}  {path}  {signature}` (no trailing summary)
   - Keep the feature gate suffix (it's important metadata)
   - Adjust column rendering to not include summary

2. **Update `render_list_recursive` in `src/render/list.rs:55-90`:**
   - Same change: no summary in default recursive output

3. **Update the search result output** in `src/lib.rs:420-455`:
   - Search results should still show summaries (they help identify the right item)
   - This is separate from the list/recursive tier system

4. **DO NOT touch `render::text::render_text`** — the default single-item view (querying a specific function/struct) should still show full docs. This change only affects list/recursive table output.

5. **Update snapshots:** Many snapshots will change. Use `INSTA_UPDATE=always cargo test`.

6. **Update affected tests** that assert on summary content in list output.
