title: "Fix duplicate output in recursive source mode (-rs/-rps)"
model: Opus
completed: true
---

When running grox -rs (or -rps), modules are rendered with their full source, then each child item (struct, enum, fn) is rendered again individually with its source. The module source already contains all children — this is pure duplication. The -p flag is orthogonal; the bug is the -rs combination.

**Root cause:** collect_children_recursive() in src/render/mod.rs:89-124 includes both modules and their children in the result list. In summary mode (-r) this is fine, but in source mode (-rs) the module's source block encompasses all child source blocks.

**Fix:** In src/render/dispatch.rs ~line 70, after collecting items via collect_children_recursive(), filter out module-kind items before rendering source:

  items.retain(|item| item.kind != ItemKind::Module);

This removes module containers from source rendering since their source is redundant with their children's source. Only affects the recursive+source code path; -r summary mode is unchanged.

**Verify:** Run grox -rs on a crate with modules and confirm no duplicate source blocks.
