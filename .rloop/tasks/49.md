## Remove the -l/--list flag from groxide CLI

**Plan reference:** docs/plans/2026-03-05-cli-redesign-implementation.md — Phase 1, Task 1

The default module/crate view already lists children. The -l flag provides a "slightly different list format" which is redundant and hard to explain. Remove it entirely.

### What to do

1. **Remove the `list` field from `Cli` struct** in `src/cli.rs:35-37`. Delete the field and its attributes.

2. **Remove `"list"` from all `conflicts_with_all` arrays** on other flags in `src/cli.rs` (source, search, impls, readme).

3. **Remove the `-l` example** from HELP_EXAMPLES in `src/cli.rs:11`.

4. **Remove list dispatch in `handle_output`** in `src/lib.rs`:
   - Remove the `cli.list` branch at ~line 724-726
   - Remove the `cli.list` branch at ~line 749-751 (ambiguous case)

5. **Update integration tests:**
   - Remove tests that use `--list` or `-l` in `tests/cli_happy.rs` and `tests/cli_errors.rs`
   - Remove associated snapshot files from `tests/snapshots/` (any snapshot with "list" in the name)
   - Update any conflict tests that reference `list`

6. **Do NOT remove `src/render/list.rs`** — the list rendering functions are still used by `-r` recursive mode (`render_list_recursive`). Only remove the `render_list` entry point function and `collect_list_items` if they become unused.

7. **Update snapshots:** `INSTA_UPDATE=always cargo test`
