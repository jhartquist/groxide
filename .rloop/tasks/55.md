## Implement docs rendering (full docs per item)

**Plan reference:** docs/plans/2026-03-05-cli-redesign-implementation.md — Phase 2, Task 7

Docs mode shows kind + name + signature + full rendered docs. For single items, this is essentially the current default view. The key change is in recursive mode where each item gets its full docs rendered (not just a one-line summary).

### What to do

1. **For single-item/container default view:** The current default text renderer already shows full docs. When `cli.docs` is set, the behavior should be the same as the current default. Wire `cli.docs` to use `render::text::render_text()` in the dispatch.

2. **For recursive mode with -d:** Create a new rendering function that outputs each item with full docs. Options:
   - Add `render_list_recursive_with_docs` to `src/render/list.rs`, or
   - Create a new function in a suitable render module

   Format per item in recursive mode:
   ```
   fn  groxide_test_api::add
       pub fn add(a: i32, b: i32) -> i32

       Adds two numbers together and returns the result.

   struct  groxide_test_api::GenericStruct
       pub struct GenericStruct<T>

       A generic struct that wraps a value of type T.
       [full docs here...]
   ```

3. **Wire into dispatch** in `src/lib.rs`:
   - In the recursive block: check `cli.docs`, call the docs-mode recursive renderer
   - In the default block: `cli.docs` should produce same output as current default (no change needed, just ensure it doesn't hit the brief path)

4. **Write integration tests** in `tests/cli_happy.rs`:
   - `docs_mode_function`: `grox -d groxide_test_api::add` — snapshot
   - `docs_mode_recursive`: `grox -r -d groxide_test_api::containers` — snapshot (should show full docs per item)
   - Verify docs content appears (not just one-line summary)

5. **Update snapshots:** `INSTA_UPDATE=always cargo test`
