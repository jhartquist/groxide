## Source mode should include rendered docs above source code

**Plan reference:** docs/plans/2026-03-05-cli-redesign-implementation.md — Phase 2, Task 8

Currently -s shows only source code. Change it to show rendered documentation above the source, making -s a strict superset of -d. The user should see the docs (rendered prose) then the source code below.

### What to do

1. **Modify `render_source` in `src/render/ambiguous.rs`:**
   - Before the source code block, prepend the item's rendered documentation
   - Use `strip_markdown` on `item.docs` to get clean text
   - Separate docs from source with a blank line and a divider or header

   Example output:
   ```
   fn groxide_test_api::add
   pub fn add(a: i32, b: i32) -> i32

   Adds two numbers together and returns the result.

   ── Source: src/lib.rs:42-44 ──
   pub fn add(a: i32, b: i32) -> i32 {
       a + b
   }
   ```

2. **Also update `render_source_ambiguous`** to include docs for each item.

3. **Write integration test** in `tests/cli_happy.rs`:
   - `source_mode_includes_docs`: `grox -s groxide_test_api::add` — verify output contains both doc text AND source code
   - Update existing `source_mode` and `source_struct` snapshots

4. **Update snapshots:** `INSTA_UPDATE=always cargo test`
