Add accessor methods to DocIndex and migrate callers away from direct field access.

## Problem

`DocIndex` fields (`path_map`, `name_map`, `suffix_map`, `items`, `trait_impls`) are `pub(crate)` and accessed directly throughout the codebase. This makes it hard to change the index structure without updating every call site.

## Fix

1. Add accessor methods to `DocIndex` in `src/types.rs`:
   ```rust
   impl DocIndex {
       /// Returns item indices matching the exact path.
       pub(crate) fn lookup_by_path(&self, path: &str) -> &[usize] {
           self.path_map.get(path).map_or(&[], Vec::as_slice)
       }
       /// Returns item indices matching the item name.
       pub(crate) fn lookup_by_name(&self, name: &str) -> &[usize] {
           self.name_map.get(name).map_or(&[], Vec::as_slice)
       }
       /// Returns item indices matching the path suffix.
       pub(crate) fn lookup_by_suffix(&self, suffix: &str) -> &[usize] {
           self.suffix_map.get(suffix).map_or(&[], Vec::as_slice)
       }
       /// Returns trait implementations for the given item index.
       pub(crate) fn trait_impls_for(&self, item_idx: usize) -> &[TraitImpl] {
           self.trait_impls.get(&item_idx).map_or(&[], Vec::as_slice)
       }
   }
   ```
2. Migrate callers in `query.rs`, `search.rs`, `render/`, `lib.rs` to use these methods
3. Keep fields as `pub(crate)` for now — this is a gradual migration, not a breaking change
4. In a future task, fields can be made private once all access goes through methods

## Important

- Do NOT change field visibility in this task — just add methods and migrate callers
- Do NOT change the `get()` method that already exists (it returns `&IndexItem` by index)
- Index building code (`index_builder.rs`) still needs direct field access for construction — that's fine
