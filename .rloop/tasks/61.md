## Support -r flag in workspace mode (recursive across all crates)

**Plan reference:** docs/plans/2026-03-05-cli-redesign-implementation.md — Phase 4, Task 14

When -r is used at a workspace root with no path argument, recursively list all items from all workspace crates. All detail flags compose: -r -b for names only, -r -d for full docs, -r -s for full source dump.

### What to do

1. **Update `handle_workspace` in `src/lib.rs`** to handle the recursive flag:
   - When `cli.recursive` is set, for each crate:
     1. Resolve crate root
     2. Call `collect_children_recursive` to get all items
     3. Apply kind filter if set
     4. Render based on detail tier (-b, -d, -s, or default)
   - Separate each crate's output with a crate header and blank line

2. **Ensure all detail tiers work in workspace+recursive mode:**
   - `grox -r`: all crates, all items, signatures (default)
   - `grox -r -b`: all crates, all items, names only
   - `grox -r -d`: all crates, all items, full docs
   - `grox -r -s`: all crates, all items, docs + source

3. **Handle JSON output** (`-j`): workspace+recursive+json should emit JSON Lines for all items across all crates.

4. **Progress feedback:** When indexing multiple crates, show progress on stderr:
   ```
   [grox] Building index for rloop_core 0.1.0... done (1.2s)
   [grox] Building index for rloop_db 0.1.0... done (0.8s)
   ```
   (This may already happen naturally from `load_or_build_index`)

5. **Test manually** against the rloop workspace:
   ```bash
   cd ~/projects/rloop/rloop_v5
   grox -r -b   # Should show all items from all 8 crates
   ```
