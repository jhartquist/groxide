Replace repeated if/else chains for output format selection with an OutputMode enum.

## Problem

Three+ places have the same pattern:
```rust
if cli.json { render_json(...) }
else if cli.brief { render_brief(...) }
else if cli.docs { render_docs(...) }
else { render_text(...) }
```

This appears in `handle_output()`, `render_recursive()`, and `handle_search()`.

## Fix

1. Define in `src/cli.rs`:
   ```rust
   #[derive(Debug, Clone, Copy, PartialEq, Eq)]
   pub(crate) enum OutputMode {
       Json,
       Brief,
       Text,
   }

   impl Cli {
       pub(crate) fn output_mode(&self) -> OutputMode {
           if self.json { OutputMode::Json }
           else if self.brief { OutputMode::Brief }
           else { OutputMode::Text }
       }
   }
   ```
2. Update dispatch sites in `render/dispatch.rs` to use `match cli.output_mode()` instead of if/else chains
3. This is NOT a trait hierarchy — just an enum for centralized dispatch

## Important

- `--docs` is a detail tier modifier, not an output mode — it composes with text output. Don't treat it as a separate mode.
- The enum should reflect format choices (json vs brief vs text), not detail levels
- All snapshot tests must pass. Run `INSTA_UPDATE=always cargo test` if output changes
