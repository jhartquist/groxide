## Implement recursive + source mode (dump everything)

**Plan reference:** docs/plans/2026-03-05-cli-redesign-implementation.md — Phase 2, Task 9

The "dump everything" mode. When both -r and -s are set, recursively traverse all items and show docs + source for each. This is the mode that turns an entire crate into one long text output.

### What to do

1. **Add recursive+source handling** in `src/lib.rs`:
   - In `handle_output`, when `cli.recursive && cli.source`:
     1. Collect all items via `render::collect_children_recursive()`
     2. For each item, call `read_source_content()` to get source
     3. Render each item with docs + source (reuse `render_source` from ambiguous.rs)
     4. Concatenate with section separators between items

   ```rust
   if cli.recursive && cli.source {
       let items = render::collect_children_recursive(using_index, effective_idx, cli.private);
       // ... render each with source
   }
   ```

2. **Consider source availability:** Not all items will have source spans (e.g., re-exports, synthetic items). Skip source gracefully when unavailable — just show docs + signature.

3. **Handle the `source` CrateSource** — `read_source_content` needs the CrateSource to find files on disk. Pass it through to the recursive rendering.

4. **Write integration tests** in `tests/cli_happy.rs`:
   - `recursive_source_module`: `grox -r -s groxide_test_api::containers` — snapshot
   - `recursive_source_crate_root`: `grox -r -s groxide_test_api` — verify it contains source from multiple modules
   - Verify source code appears for multiple items

5. **Also implement -r -b and -r -d dispatch** if not already done:
   - `-r -b`: dispatch to `render::brief::render_brief_recursive`
   - `-r -d`: dispatch to docs-mode recursive renderer
   - `-r` alone (no detail flag): use current default (signatures + summaries via `render_list_recursive`)

6. **Update snapshots:** `INSTA_UPDATE=always cargo test`
